Future Term Plugin Ideas ​

Here are some ideas for "meaty" Term Plugins that tackle difficult temporal problems, making them excellent candidates for a premium/enterprise offering.

1. Tempo SLA (Business & Working Hours Engine) ​

Standard time is linear, but business time is fragmented. This plugin would allow users to calculate deadlines and durations that skip over weekends, public holidays, and non-working hours.

  • The "Meat": Building an engine that can parse customized holiday calendars, handle time zones (e.g., 9-5 in New York vs 9-5 in London), and calculate "business hours" across multi-day spans.
  • API Examples:
    typescript
    // Add 12 business hours (might span across a weekend or holiday)
Tempo().plus({ businessHours: 12 }); 

// How many working days until the deadline?
Tempo('2024-12-20').until('2025-01-05', 'businessDays');
  • Why it's great: Every B2B SaaS, helpdesk (Zendesk-style), and logistics company needs this. It's highly valuable for a commercial tier.

2. Tempo Markets (Financial Settlement & Trading Cycles) ​

Financial systems rely on highly specific trading calendars. This plugin would be aware of global stock exchanges, early-close days, and settlement rules.

  • The "Meat": Maintaining or dynamically fetching market holiday lists (NYSE, LSE, ASX, etc.). Calculating T+2 settlement dates, quarterly earnings boundaries, and tracking whether the market is currently "open" or in "after-hours".
  • API Examples:
    typescript
    // Calculate a T+2 trade settlement date for the NYSE
Tempo().term('settlement_date', { market: 'NYSE', offset: 2 });

// Check if a timestamp is within active trading hours
Tempo().is('market_open', { market: 'ASX' });

3. Tempo Roster (Complex Shift & Rotation Patterns) ​

Workforce management deals with complex, rotating cycles that don't fit neatly into weeks or months (e.g., the "Panama Shift" pattern, 4-days-on / 4-days-off, or night shifts that span across midnight).

  • The "Meat": Building an anchor-based cyclic engine. If a nurse started a 28-day rotating schedule on Jan 1st, calculating what shift they are on in 8 months is computationally heavy.
  • API Examples:
    typescript
    // Define a cyclic rotation 
const panamaShift = Tempo.Rotation({ pattern: [2, 2, 3, 2, 2, 3], anchor: '2024-01-01' });

// What shift type are they on next Tuesday?
Tempo('next tuesday').term('shift_type', { rotation: panamaShift });
